# Content Creation — Protocol Registry

| Protocol | Version | Status | Cross-Dept | Description |
|----------|---------|--------|-----------|-------------|
| content-request | v1.0 | active | yes — marketing | Governs how content requests flow from Marketing into Content Creation and back |
| brand-voice | v1.0 | planned | no | Internal guidelines for maintaining consistent brand voice across all writers |

## How to Add a Protocol

1. Create `protocols/{name}.md` with YAML frontmatter (name, version, status, owner, cross-dept list, last-updated)
2. Add row to this registry
3. If cross-dept: also add to `runbooks/protocol-registry.md`
4. Cross-dept changes require both dept heads to sign off
