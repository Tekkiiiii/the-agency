# Testing Department

**Call this department when you need to verify quality before shipping** — QA testing, screenshot evidence, accessibility audits (WCAG/screen readers), API testing, performance benchmarking, test result analysis, tool evaluation, and workflow optimization. Also for blocking fantasy approvals with evidence-based certification.

**Leader**: Reality Checker
**Sub-groups**: validation | analysis | performance
**Model tier**: Members = Sonnet, Leaders = Opus

## Members

| Agent | What it does |
|---|---|
| Reality Checker | Evidence-based certification, blocks fantasy approvals, "NEEDS WORK" default |
| Evidence Collector | Screenshot-obsessed QA, finds 3-5 issues with visual proof |
| Accessibility Auditor | WCAG audits, assistive technology testing, inclusive design |
| API Tester | API validation, performance testing, third-party integration QA |
| Performance Benchmarker | System performance measurement and improvement |
| Test Results Analyzer | Quality metrics, test result evaluation, actionable insights |
| Tool Evaluator | Software/platform assessment and recommendation |
| Workflow Optimizer | Process improvement, automation of business workflows |

## Skills

These skills are loaded as process gates by dept members:
- **qa** — systematic QA testing and bug fixing
- **qa-only** — report-only QA without fixes
